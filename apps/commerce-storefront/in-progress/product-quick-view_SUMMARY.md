## Architecture

The implementation follows the **PWA Kit extensibility override pattern**:

```
overrides/app/components/product-tile/index.jsx
  └─ imports base ProductTile from @salesforce/retail-react-app
  └─ imports QuickViewModal via relative path (../quick-view-modal)
  └─ wraps tile in Box with hover state → shows "Quick View" button
  └─ on click → opens QuickViewModal with product ID

overrides/app/components/quick-view-modal/index.jsx
  └─ Chakra Modal + useProductViewModal hook
  └─ Commerce SDK React hooks for product data fetching
  └─ Variant selection + add-to-cart flow
```

**Key design decision:** `QuickViewModal` is imported via **relative path** (`../quick-view-modal`), not the `@salesforce/retail-react-app` alias, because it is a new custom component not present in the base template. The PWA Kit override resolver only creates aliases for files that exist in the base package.

| Modal closes correctly | ❌ Timeout | Same root cause |
| Quick View does not navigate away from PLP | ❌ Timeout | Same root cause |

**Root cause:** The Playwright E2E tests use `page.waitForLoadState('networkidle')`, which never resolves because PWA Kit's Hot Module Replacement (HMR) maintains a persistent WebSocket connection during dev mode. This is a **test infrastructure issue**, not an application bug.

**Recommended fix:** Replace `waitForLoadState('networkidle')` with `waitForLoadState('domcontentloaded')` or use explicit element selectors in `e2e/quick-view.spec.ts`.
